## 明白自身需求

事实上，AI 的用法多种多样，而不同人使用 AI 有不同的需求或目的。由于本人是 OIer，因此使用方案可能不够全面。但大体来说，AI 的使用方式有：

- 文本类：
    - 编写、修改文章（文章理解与创造力）
    - 编写、重构工程项目（代码编写能力与理解能力、长上下文、指令尊崇度）
    - 学术交流，包括各种科目（世界知识水平、推理能力）
- 艺术类：
    - 图像生成（创造力和逻辑严谨性、指令尊崇度）
    - 视频生成（同“图像生成”）
    - 音频生成：
        - 模拟人声
        - 歌曲创作（创造力）

同时，你应该了解自身能够承受多少的支出。不过目前大多数人对于 AI 仍旧处于观望的态度，因此后文我会介绍一些相对强力的免费 AI 使用方式。

另外，如果你只是想初步玩玩 AI，对 AI 依赖度不高，没有专门的、清晰的需求，不愿意进行复杂的配置，也不追求前沿模型。那么 [豆包](https://www.doubao.com/) 是你的最优选择，没有之一，不用往下看了。

## 前置知识

充分了解的可以直接跳过。

### 基本常识

#### 模型与 API 相关

- **Model（模型）**。
- **Message（对话历史）**：包含用户输入和先前 AI 的输出。
- **System Prompt（系统提示词）**：给系统的全局指令，可以决定 AI 输出格式。
- **Token（词元）**：相当于 AI 的特殊字符编码，一个 Token 表示一个/多个字符，组合而成就拼成了 AI 的输入和输出。
- **Max Tokens**：最大输出 Token 数量。
- **Context Windows（上下文窗口）**：一般以 Tokens 计。表达模型能够容纳的上文 Tokens 数量。此数值越高不一定越好，还需要检查上下文记忆能力（如 Gemini 全系标配 1M 上下文但记忆效率明显不如 Claude 系列）。
- **Temperature（温度）**：输出的随即程度，越高越发散。
- **Top P（核采样）**：只从累积概率达到 p 的候选 token 中采样。
- **Stream（流式输出）**：是否逐 Token 输出。

#### 功能特性

- **Vision（视觉）**：是否支持图片输入。
- **Function Calling（函数调用/工具使用）**：是否通过自定义指令调用工具。
- **Structured Output（结构化输出）**：按照 JSON 格式输出。
- **Multimodal（多模态）**：支持多种模态输入，如音频、视频等。

### 使用方式

了解主流的使用 AI 的方式。

#### 官方订阅制

##### 国内 AI

大多需要一个能够正常使用的 +86 手机号，有些同样支持邮箱。

如果需要订阅服务，大多微信或 QQ 即可使用。

##### 国外 AI

首先需要一个支持非香港、澳门、台湾节点的 VPN^[1]。推荐使用新加坡或美国西部节点，延迟低且解锁范围广。

如果需要订阅服务，请在网上购买虚拟卡（国内电商平台即可购买，注意查看汇率，选稳定性高、汇率低的。购买前联系客服查看对国外 AI 订阅服务的支持情况）。

需要注意自身 IP 的纯净度，选稳定的 IP，不要乱改节点。

^[1]: 目前国外 AI 对于中国地区的管控非常严格，港澳台地区仍然无法使用。

#### API 调用制

> [!attention] 注意
> 
> API 调用制大多无法使用官方方便的对话服务，需要使用第三方软件调用或自行编写代码调用。但是可带来更多灵活性（使用更多第三方软件的附加功能）。
> 
> 另外，如果使用量较大，API 调用的费用一般比订阅制贵不少。但是对于使用量不稳定的人群，使用 API 可以节省成本。

##### 国内/外 官方 API

一般较贵。国外 API 同样需要 VPN，大多还需要虚拟卡。个人用户不推荐。

##### 中转 API

**这是一个巨大的坑**。*一定要自己辨别之后再下单*。

中转 API 顾名思义，就是通过中转服务器将对服务器的 API 调用中转到官方 API 当中。实则不然，有些是模拟 API 调用。它在官方网站处注册大量的订阅制账号，随机依靠订阅制账号便宜的优势模拟 API 调用给用户用，自然卖的比较便宜。（特别突出的就是 Claude 系列模型，便宜的几乎全是订阅账号使用 Claude Code 并模拟 API 调用）也有白嫖官方 API 免费额度的。

但是，这种方式非常不稳定。官方免费额度政策一旦改变，或是开始大面积封杀此类账号，中转站的服务就会突然不可用，或是高度抬价。

另外，中转站的后端是可以修改模型路由的，很有可能你调用高级模型，钱也是按照高级模型算，但是

因此，一定要选择口碑良好、较稳定的中转站，但是这种一般价格本身就很贵。如 [OpenRouter](https://openrouter.ai/)，本身价格就与上游厂商同步了。不过其较为稳定且无需 VPN，还有部分免费模型。

## 模型选择

在明白了自身需求、学习了相关知识之后，就该进入模型选择了。这可能将直接关系到你最终 AI 的使用方式和使用感受。

### 文本类

#### “御三家”

无论跑分还是社区知名度均位列前三名的三家不同美国公司 Anthropic、Google 和 OpenAI，其发布的模型系列名称分别为 Claude、Gemini 和 GPT^[2]。除了使用中转站以外，御三家的所有服务均需要搭配 VPN。

跑分数据来源于 [Arena AI](https://arena.ai/)，同时结合了我的个人使用体验。

^[2]: OpenAI 发布的文本模型名字叫 GPT，但是 AI 应用叫 ChatGPT。Claude 和 Gemini 则是模型与应用名称相同。

##### Claude

Anthropic 是由 OpenAI 的前员工离职后创建的 AI 公司，前期重点研究 AI 代理编码。MCP 协议、Skills 等都是它搞出来的。

目前 Claude Opus 4.6 是文本处理跑分当中分数最强的 LLM，也是我实际体验中最好的 AI。因为 Anthropic 的调教，Claude 在回复的时候不会献媚，写代码也都在干实事。虽然大部分时候还是用的 256K 上下文（1M 上下文太贵了），但是它的记忆能力确实很强。

因此 Claude 确实是现阶段文本处理方面最强的 AI，也可能是这家公司完全不搞图像生成或视频生成吧。**唯一的缺点是太贵且不支持图像/视频生成**。中端 Sonnet 系列 API 价格已经向别家的旗舰看齐了。

Claude 的幻觉率压得很低，这使得它更适合需要严谨思维的工作，如写代码。

- [b] **Claude 的最佳使用方式是官方订阅**。

虽然 API 价格贵，但是并不妨碍 Claude 的订阅性价比很高。**注意 Claude 订阅最低档次就是 20/月**，起步价相比于 **ChatGPT** 和 **Gemini** 更贵。

有人逆向计算过，若榨干额度，Claude Pro 订阅使用量是 API 的 9+ 倍，Claude Max (5x/20x 都一样) 是 API 的 13+ 倍。这个数据相当恐怖，因此个人使用完全没有理由去购买 API。

因为 Claude 订阅的用量计算最特殊，用量与使用 Tokens 是成正比的，而 Gemini 和 OpenAI 按照消息数计算。

两种计算方式没有明显的优劣之分，但是我个人更偏向于 Claude。对于简单、多消息的任务，使用 Claude 不会明显地消耗用量。

##### Gemini

Google 开发的大模型，用途最广。

最新的是 3.1 系列，有 Flash (Lite) 和 Pro 系列。前者性价比高、速度快，后者推理能力强。对比 Claude，有很明显的性价比优势，毕竟旗舰模型 Pro 价格甚至比 Claude Sonnet 低。Flash 系列更是在该价位无敌。

Gemini 3.1 Pro 的文本处理能力仅次于 Claude Opus 4.6，而且全系内置实时 Google Search（内网神力）。缺点是幻觉率还是偏高。

全系支持 1M 上下文，但是记忆能力差。**能存下但记不住**。

- [b] Gemini 的最佳使用方式是 API 调用。

因为 Google 太慷慨了，只需要在 Google Cloud Platform 中验证支付方式并支付十美元，你就有 90 天的 300$ 免费配额使用，而且后期大概率还会经常给你送免费额度。

相比起来，Gemini 应用就有些拉了，不仅提示词当中显示思考链长度，而且相关工具也不如 Claude 全面。

##### GPT

*昔日的神，终究还是陨落了*。

作为最早整出真正可用 LLM 的公司，曾经喊道要实现 AGI，但目前已被 Claude 和 Gemini 之流尽数超越。

代理编码被 Claude 肘击，科学推理被 Gemini 肘击，自身也无成本优势。API 额度送的没有 Gemini 多，订阅制没有 Claude 使用量大。唯一拿得出手的就是应用交互体验。

不过瘦死的骆驼比马大，GPT 模型的水平还是不错的，正好在 Arena AI 上位列第三。从昔日的冠军变为了御三家的吊车尾。

##### 闲话

之前 Anthropic 将中国列为敌对国家，在网上掀起了轩然大波。不过仔细想想，御三家从一开始就没有像中国提供过服务过（港澳台地区曾经能用，现在和大陆一样了）。首先排除公司不想赚钱，因为中国实际上 AI 需求量是很大的。

很明显只是美国不想让中国的 AI 发展起来而已。事实上目前中国的 AI 哪怕是顶级旗舰模型就是不敌御三家，性价比再高也没有用。尽管中国 AI 多次接近过御三家的顶尖模型，但是从未真正超越过。希望未来中国能够训练出能力真正超过御三家的 AI，但是一定不要定价过高，否则就会成为下一个垄断的“御三家”。

说了这么多，不过我们消费者还是不要有太大心理负担。哪个强用哪个，这才是真正的道理。至少御三家并不禁止你通过 VPN 访问（明里暗里还是想赚钱的），封号也只是封中转站的机器人账号。

#### 国产下位替代

这一部分就比较简略了。得益于定价便宜、没有试错成本，大家可以自行使用、体验一下。

##### Qwen（千问）

**优势**：生态广；开源可部署。

各个方面性能都较为平衡。

##### Kimi（Moonshot AI 月之暗面）

**优势**：Agent 能力强、写作能力强、多模态能力强。**缺点**：幻觉率高。

##### GLM（智谱 AI）

**优势**：Agent 能力强、工具调用能力强。

#### 补充

##### Grok

文本评分当中出乎意料地超越了 OpenAI，但是其它项目很拉了。配合 X（Twitter）使用效果俱佳，但是 X 上面本身有很多涉黄涉政的。Grok 因为吸取了 X 的数据集，可以生成黄色文本和图片。

这点一定要小心！不然某天警察同志就给你抓去了。

##### DeepSeek

去年 1 月 DeepSeek R1 推理模型横空出世直接对标 OpenAI O1 模型，但成本低太多，被吹上天，现在沉寂了。大概是在股市里捞够钱了吧。

旗舰模型已经很久没更新了，能力已经跟不上时代了，而且 Tokens 输出速度极慢，定价再低也没有用。

*不过疑似 V4 新模型即将发布，到时候看具体水平再说*。

## OIer 模型推荐

### 有钱的

首先应该先整个梯子。

优先上 Claude 订阅，Pro 就够了。不要听网上人说 Pro 额度一下就用完了，只要不用 Claude Code 哪怕一直用 Opus 调题 5hr 额度也用不完。也就每周配额可能会触发限制。

Qwen 和 GLM 这俩侧重点分别在本地部署和工具调用上，对 OI 影响不大。
